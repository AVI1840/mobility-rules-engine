# מנוע זכויות ניידות — כרטיס פרויקט לפורטל AI

---

## הבעיה הארגונית — ביטוח לאומי

המוסד לביטוח לאומי מנהל עשרות תחומי זכויות: נכות, ניידות, סיעוד, הבטחת הכנסה, ילדים, שאירים, אמהות ועוד. כל תחום נשען על מאות ואלפי מקורות מקצועיים שהצטברו לאורך עשרות שנים:

- **אלפי חוזרים מנהליים** שמתעדכנים באופן שוטף
- **תדריכים מקצועיים** שמנחים את הפקידים בשטח
- **הסכמים ותיקוני חקיקה** שמשנים את תנאי הזכאות
- **פסקי דין** שיוצרים תקדימים ומשנים את הלוגיקה הקיימת
- **חוות דעת משפטיות** שמפרשות מצבי גבול
- **הגדרות מחשבונים** שמתרגמות כללים לחישובים

**הקושי בשטח:**

הפקיד נדרש לדעת, לאתר, לפרש ולהחיל את כל אלה — בזמן אמת, על כל מקרה, בצורה עקבית. זה לא ריאלי. התוצאה:

| אתגר | ביטוי בשטח |
|-------|------------|
| עומס ידע | פקיד חדש לוקח שנים להגיע לרמת מומחיות |
| קושי בהדרכה | קשה ללמד אלפי כללים בהכשרה |
| פערי יישום | חוזר חדש יוצא — לא תמיד מיושם מיד ובאופן אחיד |
| חוסר אחידות | אותו מקרה — החלטות שונות בין פקידים ובין סניפים |
| קושי בהטמעה | שינוי מדיניות דורש הדרכה, תרגול, ובקרה — תהליך ארוך ויקר |
| ביקורת מוגבלת | קשה לנמק ולתעד כל החלטה בצורה שניתנת לביקורת |
| זכויות שנפספסות | אזרחים לא מקבלים את מה שמגיע להם |

---

## הפתרון — גישת Rules-as-Code

במקום לנסות ללמד את הפקיד את כל הכללים — **מקודדים את הכללים עצמם** ומאפשרים למנוע לבדוק אותם אוטומטית.

```
חוזרים + הסכמים + פסקי דין + תדריכים
              ↓
      כללים מכונתיים דטרמיניסטיים
              ↓
   פקיד מזין נתונים → מקבל המלצה מנומקת
              ↓
      פקיד מחליט + נותן פידבק → שיפור מתמיד
```

הפקיד לא צריך לזכור את כל החוזרים. הוא מזין נתונים ומקבל המלצה עם ציטוט מדויק למקור.

## ה-MVP — תחום הניידות

**למה ניידות ראשון?**
- תחום מוגדר עם מקורות ידועים
- קיים תדריך מקצועי מקיף
- קיים הסכם ניידות מפורט עם תיקונים
- קיים מחשבון פעיל באתר החיצוני — ניתן ללמוד מההגדרות שלו
- לוגיקה יחסית ברורה — מתאים לפיילוט ראשון

**מקורות שמקודדים במנוע:**

| סוג | מקורות |
|-----|--------|
| הסכם | הסכם הניידות (כולל תיקון 24), הסכם קרן הלוואות |
| תדריך | תדריך ניידות מקצועי |
| חוזרים | 1810, 1905, 1931, 1936, 1984, 1996, 2056, 2132 |
| פסקי דין | שושנה לוי, שקד ארועטי, גלית לביא, רות הדאיה |
| חוות דעת | המשך תשלום קצבה 3 חודשים |
| מחשבון | הגדרות המחשבון הפעיל באתר החיצוני |

---

## איך עובד

1. **פקיד מזין נתוני מקרה** — אחוז נכות, מצב תפעולי, סוג תביעה
2. **המנוע בודק כללים רלוונטיים בלבד** — לפי סוג התביעה, לפי היררכיה משפטית
3. **פקיד מקבל המלצה** — זכאי/לא זכאי + הנמקה בעברית + ציטוט משפטי מדויק + רמת ודאות
4. **פקיד מחליט ונותן פידבק** — לייק/דיסלייק + הערה → נשלח לדשבורד פידבקים

## מודל ודאות — שלוש רמות

| רמה | משמעות | פעולה |
|-----|--------|-------|
| ✅ דטרמיניסטי | כל התנאים ברורים, אין עמימות | ניתן לאוטומציה בעתיד |
| ⚠️ המלצה | בסיס משפטי חזק, דורש אישור | פקיד בכיר מאשר |
| 🔍 שיקול דעת | דורש סקירה מקצועית | סקירה ידנית |

---

## השיטה

- **Rules-as-Code** — חוזרים ופסקי דין הופכים לכללים מכונתיים
- **היררכיה משפטית** — פסיקה > חקיקה > חוזר > נוהל
- **Human-in-the-Loop** — המנוע ממליץ, הפקיד מחליט
- **Pilot-First** — ולידציה מקצועית לפני כל הרחבה
- **ללא AI גנרטיבי** — לוגיקה דטרמיניסטית בלבד, אפס הזיות
- **פידבק מובנה** — כל הרצה ניתנת לדירוג ולהערה

## הדרך קדימה

```
שלב 1 (נוכחי): MVP ניידות — כלי תומך החלטה + פידבק
שלב 2: ולידציה על מדגם מקרים היסטוריים
שלב 3: הרחבה לתחומים נוספים (נכות, סיעוד, הבטחת הכנסה)
```

ניידות הוא ה-MVP. אם הפיילוט יוכיח ערך — אותה שיטה מיושמת על כל תחום. **מפעל מנועי מדיניות** לכל ביטוח לאומי.
